A specimen of textpour

Pour text into
any shape.

A line breaker like Pretext fills one line at one width. The loop that pours text into a shape on top of it is about twelve trivial lines you could inline. The one reusable thing textpour really is: turning any 2-D shape — holes, concave polygons, glyphs, photo masks, boolean combinations — into correct per-row spans. These demos isolate exactly that, and own up to what you don't need a library for.

the whole idea, in one loop
// Pretext breaks the lines. This loop is the trivial glue.
let cur = source.start();
for (let y = top; y + lh <= bottom; y += lh) {
  const spans = region.spansAt(y + lh/2);  // ← the only part you'd rewrite, per shape
  for (const [x0, x1] of spans) {        // 1..N disjoint pieces on this row
    const line = source.nextLine(cur, x1 - x0);
    if (!line) break;
    draw(line.text, x0, y + ascent);
    cur = line.end;                       // thread one cursor across the gaps
  }
}

Every demo runs the real kernel from ../dist/src/index.js with Pretext pulled from esm.sh — served over http (ES modules need it). The geometry is computed in pure JS; Canvas2DRenderer paints it. Most of what you see is one function — region.spansAt(y) — for shapes Pretext has no concept of.
