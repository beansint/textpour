The finding · the cursor trick

One line, many gaps.

A line breaker like Pretext fills one line at one width. Drop an obstacle into the middle of a column and that model breaks — the line is now two pieces with a hole between them. textpour threads a single reading cursor through every span on the row, so a line flows around islands on both sides at once. Drag the stones.

The region is subtract(rect, …circles); the islands are real holes, so each scanline returns several disjoint spans. With multiSpan:'fill' one cursor is carried across them. The multi-span rows and max spans on a row are impossible in a one-width-per-line model.

raw Pretext, by hand
// the loop never changes. spansAt(y) is the work:
function spansAt(yc) {
  let spans = [[x, x + w]];             // the column at this row
  for (const c of circles) {            // subtract each island
    const dy = yc - c.cy;
    if (Math.abs(dy) >= c.r) continue;
    const h = Math.sqrt(c.r*c.r - dy*dy);
    spans = subtractInterval(spans, c.cx - h, c.cx + h);
  }
  return spans;                         // 1..N disjoint pieces
}

The loop never changes. spansAt is boolean interval subtraction of N circles from a rect — re-derivable, but this is the piece textpour hands you (subtract() / CompositeRegion).
