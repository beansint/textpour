The finding · multi-span · mask region

A letter made of words.

Pretext lays out one line at one width. So it cannot pour a single line through the two halves of an A, or around the bowl of an &. textpour turns the glyph into a region and threads one cursor across every disjoint span on each row — the counters fill themselves.

glyph & auto-fit — px lines — multi-span rows — coverage —

Pipeline: rasterize the glyph → read its ink into a maskRegion → autoFit the body size so the words pack the shape → pour with multiSpan:'fill' so one line crosses the counters. The multi-span rows count is the bit Pretext can't produce.

raw Pretext, by hand
// loop unchanged. spansAt(y) must now rasterize + scan a row:
function spansAt(yc) {
  const row = Math.floor(yc), spans = [];
  let inRun = false, start = 0;
  for (let x = 0; x < W; x++) {                 // run-length scan
    const inside = glyphAlpha[row*W + x] >= 128;
    if (inside && !inRun) { inRun = true; start = x; }
    else if (!inside && inRun) { inRun = false; spans.push([start, x]); }
  }
  if (inRun) spans.push([start, W]);
  return spans;                                  // counters -> several spans
}
