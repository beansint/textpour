textpour maps any pixel to an exact character position and back — the basis for carets, selection, and editing in canvas-rendered text.

22px
Row — Grapheme — / — Caret x — px ← · | · →

buildPrefixWidths(lineText, measurer) returns a cumulative array of x-offsets at each grapheme boundary — measuring the prefix string each time so kerning is captured. xToGraphemeIndex(prefix, x) binary-searches that array and snaps to the nearer boundary. graphemeIndexToX(prefix, idx) reads back the pixel position. Together they give a pixel-perfect, grapheme-correct cursor for canvas text editing — emoji 👍 and accented characters land as single units, never split.

the wiring · textpour vs raw Pretext
textpour
const prefix = buildPrefixWidths(line.text, canvasMeasurer(ctx, FONT));
const i = xToGraphemeIndex(prefix, pointerX - line.x);  // pixel -> grapheme
const caretX = line.x + graphemeIndexToX(prefix, i);    // grapheme -> pixel
