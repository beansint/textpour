Motion / reflow

Prepared once. Reflows every frame.

The text is shaped a single time; drag the stone (or let it orbit) and the column re-pours around it live — no DOM, no re-shaping.

new PretextLineSource(text, font) runs once — this is the expensive Pretext prepare pass that segments and measures every word. shapeFlow(source, subtract(rect, circle(ox, oy, r)), opts) runs every frame — it reuses the prepared data and only solves the geometry for the current obstacle position. This is the core plan/paint split: shape once, reflow freely.

const region = subtract(rect(...), circle(obx, oby, r));

raw Pretext, by hand
// rect minus one circle is genuinely easy (~6 lines):
function spansAt(yc) {
  const dy = yc - oby;
  if (Math.abs(dy) >= r) return [[x, x + w]];   // row misses the stone
  const h = Math.sqrt(r*r - dy*dy);
  return [[x, obx - h], [obx + h, x + w]];       // two pieces
}

Honest one — rect-minus-a-circle spans are easy (~6 lines). The value here isn't the geometry; it's reuse plus prepare() running once while shapeFlow re-pours every frame.
